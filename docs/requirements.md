# 【要件定義書】個人向けPL管理・家計簿アプリケーション

## 1. プロジェクトの目的・コンセプト

- **完全手動入力による「肌感覚」の維持**: 自動連携（マネーフォワード等）のタイムラグやブラックボックス化を排除し、手入力することでリアルタイムな資金把握と支出への意識向上を図る。
- **個人財務のPL（損益計算書）化**: 単なる家計簿ではなく、企業会計レベルの粒度で「手取り（トップライン）」「固定費」「変動費」「最終利益（可処分所得）」を可視化する。
- **1入力・全連動のデータベース設計**: 1回の支出入力で、カテゴリ別集計から各ウォレット（銀行・クレカ）の残高、将来の引き落とし予定額まで全てリアルタイムに連動させる。

## 2. 主要機能一覧

### ダッシュボード（BIビュー）機能
- 1ヶ月分の支出・収入の俯瞰（スプレッドシートの一覧性を再現）。
- カテゴリ別の比率グラフ、前月比、予算消化率のリアルタイム表示。

### トランザクション（支出・収入）入力機能
- 日次での支出手入力（金額、カテゴリ、決済手段、メモ）。
- 収入の予実管理（概算入力から「確定フラグ」への切り替え）。
- 給与明細のOCR読み取りによる各控除項目・税金の自動入力。

### ウォレット（口座・決済手段）管理機能
- 現金、各銀行口座、各クレジットカードの残高および未払い額のリアルタイム表示。
- クレジットカードの想定引き落とし額の算出と、指定口座（三井住友銀行等）からの引き落とし実行（消込）機能。

### 資金移動（振替）機能
- 口座間の資金移動入力（手数料の自動支出計上含む）。

### サブスク・固定費マスタ機能
- 毎月の固定費の自動計上と、解約時の簡単な停止（終了年月設定）機能。

## 3. 非機能要件（今後の検討事項）

- AWSを用いたクラウドインフラ構築。
- スケールとカテゴリ増減に耐えうるリレーショナルデータベース（RDB）設計。
- モバイル（iOS等）での閲覧・入力を前提としたレスポンシブなUI設計。

## 4. 現運用の詳細（Phase 2 で反映）

実際のスプレッドシート運用をヒアリングし、具体的なカテゴリ・口座・カードの
マスタを `docs/master-data.md` に整理した。要点は以下。

- **会計年度（FY）= 4月〜翌3月**。年間でのPL振り返りを重視する。
- **タブは3種**: 「日次（変動費のみ1円単位で入力）」「月次（支出：固定費＋変動費）」
  「月次（収入：給与・控除・その他収入）」。
- **変動費は3〜4階層のカテゴリ**で管理し、取引は最下層（葉）に入力、上位は自動集計。
- **固定費は定額をFY分仮置きし、解約時は0円＋グレーアウト**（＝マスタの終了年月セット）。
- **クレカは3枚（EPOS / PayPay / Olive）、全て三井住友銀行から引き落とし**。締め日・支払日は master-data.md 参照。
- **給与明細は金額項目に加え、労働時間・時給換算などのKPIも管理対象候補**。

詳細・確定前の要確認事項（🔶）は `docs/master-data.md` を参照。
